赛题简介：介绍整个赛题的实现目标、实用价值、涉及技术和整体要求

人工智能和物联网等技术落地，势必加速全屋智能行业的发展，但同时也带来用户隐私和数据安全问题，目前国家大力推进信息技术国产化战略，以国产OpenHarmony操作系统为底座，打造国产开源生态，OpenHarmony旨在为物联网(IoT)和全场景智能设备提供统一的操作系统解决方案，支持多种硬件平台和设备类型。

【实现目标】

本赛题旨在基于OpenHarmony开发一个家居设备智能控制程序，实现远程控制设备的状态，兼顾网络协议传输安全性，同时兼顾HarmonyOS的适配。

【实用价值】

解决传统基于iOS，Android等国外操作系统的底座生态的应用在中国的垄断状态，推广国家开源鸿蒙生态应用建造，同时打造国产信创下家居设备控制的安全性，提升家居应用的安全性。

【涉及技术】

OpenHarmony、HarmonyOS、ArkTS等

【整体要求】

基于OpenHarmony操作系统开发一个家居设备智能控制系统(APP、Web、客户端等形式不限，以下统称为APP)，实现远程控制设备的状态，兼顾网络协议传输安全性，同时兼顾HarmonyOS的适配。

 

赛题业务场景：描述赛题相关的真实企业业务背景。从真实场景中，适当简化或者提炼出适合比赛的赛题场景

当前传统家居控制系统的APP主要基于iOS、Android等国外主流操作系统，普遍存在以下痛点：

1、不利于国产设备之间的协议传输，容易被“卡脖子”，造成经济损失；

2、家居控制系统非常注重隐私要求，基于国产操作系统可以很好地改善这一要求，同时可以积极建设国家开源生态的应用。

 

基本功能需求

基于OpenHarmony操作系统的APP要求可以适配HarmonyOS，控制房门开关、房间灯光开关、房间内温度与湿度的监测、房间内空调设备的开关以及温度调整等功能，同时要兼顾APP的可扩展性，可以使用仿真形式，也可以采用实物展示。

控制中心：

使用国产主板和网关组合成设备控制中心，控制中心用来接入家居设备，通过APP和网关，使得设备之间联动控制。

照明中心：

通过APP实现房间灯光开关远程控制。结合环境因素，实现人体感应控制灯的开关

温湿度控制中心：

通过预置温湿度传感器获取当前房间内温湿度环境，用APP控制房间内空调进行温度，湿度的调节，APP要兼容不同品牌空调的控制，例如：海尔、格力、美的等品牌。

智能门禁：

通过APP控制门禁开关、实现远程控制门禁(控制方式不限制，选择一个即可)。

 

非功能性需求

协议传输的安全性兼顾传输层加密、数据层加密和密钥层保护。
